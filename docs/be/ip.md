# IPv4和IPv6
## IPv4
IPv4长度是32位（bit），即一个IPv4地址由 32个二进制数字（0或1） 组成。

为了方便人类读写，这32位被分成 4段，每段 8位（称为一个字节或八位组）。
1. 每个8位二进制数会转换为一个十进制数。
2. 转换后的范围是 0 到 255
3. 最后用点（.）连接这四个十进制数。

32位二进制可以组成的唯一组合数量是2的32次方，也就是大约 43亿个 唯一地址。这直接导致了IPv4地址的耗尽危机，并催生了IPv6（128位）的出现。


## IP地址的分配与管理
IP地址不是随机出现的，它像电话号码一样，是按层级、按区域分配的。

全球分配：IANA（互联网号码分配机构）将大块的IP地址段分配给全球五大区域互联网注册机构。

区域分配：RIR（如亚太的APNIC、中国的CNNIC）再将地址段分配给其区域内的大型网络服务商（电信、联通、移动等）和大型机构。

本地分配：最终，你的网络服务商（ISP）从其拥有的地址段中，动态或静态地分配一个IP地址给你的路由器。

因此，每个IP地址块都“隶属”于某个运营商，并被记录在公开的RIR数据库中，关联到一个国家、地区甚至城市。

## IPv6
IPv6长度是128位，是IPv4（32位）的4倍。

IPv6采用冒号十六进制表示法，每16位为一组，共8组。

## DNS
DNS根域名服务器是互联网域名系统（DNS）的最高层级权威服务器，是全世界所有域名查询的“总目录”或“导航起点”。
域名就是通过遍布全球的“DNS服务器”这个特定服务，一步步查询找到对应IP地址的。
